RELATÓRIO TÉCNICO — PROMPT 15B (Otimização Autenticação JWT)
Data: 2026-05-17

1) Resumo executivo
- Implementada autenticação JWT otimizada para tokens com claims modernas (userId + roles), eliminando a necessidade de chamar `CustomUserDetailsService.loadUserByUsername(...)` em toda request.
- Introduzido `JwtPrincipal` (principal leve) + `JwtAuthenticationFactory` (monta Authentication a partir das claims).
- Mantido fallback legado para tokens antigos (configurável).
- Adicionadas properties para modo strict e validação opcional de usuário ativo via query leve (sem carregar UserDetails).
- Ajustado `TenantTokenService` para suportar principal `JwtPrincipal` (token moderno), mantendo compatibilidade.

2) Estratégia aplicada
- Token moderno: autentica por claims → menor latência e menos queries.
- Token legado: usa `loadUserByUsername` quando `consuma.security.jwt.allow-legacy-userdetails-fallback=true`.
- Strict mode opcional:
  - `strict-user-validation`: permite forçar validação conservadora
  - `validate-user-active`: valida usuário ativo por query leve (exists)
