1. Objetivo

Implementar e homologar gestao segura de senha inicial/temporaria para usuarios
provisionados pela plataforma, incluindo troca obrigatoria pelo proprio usuario,
reset por Platform Admin, bloqueio de senha temporaria expirada, propagacao de flags
em Auth/JWT e preservacao do fluxo Tenant Provisioning Access.

2. Escopo entregue

* Auth:
  - POST /auth/password/change autenticado por token global.
  - Login JWT retorna mustChangePassword, passwordResetRequired,
    temporaryPasswordExpiresAt e lastPasswordChangedAt.
  - JWT global passa a carregar claims de estado de senha.
  - /auth/password/change nao e tratado como endpoint publico.
* Platform:
  - POST /platform/users/{userId}/password/reset para reset administrativo.
  - Reset retorna senha temporaria apenas na resposta imediata.
  - Sem token retorna 401.
  - Token tenant-scoped retorna 403 no endpoint platform global.
* Tenant Provisioning Access:
  - Senha inicial gerada via servico centralizado.
  - POST /platform/tenants/{tenantId}/access/reset-password delegado ao mesmo servico.
  - GET /platform/tenants/{tenantId}/access expande flags de reset sem expor senha temporaria.
  - Cross-tenant reset retorna 404.

4. Regras de seguranca e integridade

* A senha temporaria nao e persistida em texto puro; apenas o hash BCrypt e salvo.
* Senha temporaria e retornada somente na resposta de provisionamento/reset.
* Troca propria exige token global do usuario.
* Token tenant-scoped nao pode trocar senha propria global nem executar reset platform.
* Senha atual incorreta retorna 400.
* Confirmacao divergente retorna 400.
* Senha fraca retorna 400.
* Nova senha igual a atual retorna 400.
* Senha temporaria expirada retorna 409 com code TEMPORARY_PASSWORD_EXPIRED.
* Reset admin de usuario inexistente retorna 404.
* Reset sem token retorna 401.
* Reset com token tenant em endpoint platform retorna 403.
* Resumo de acesso do tenant nao contem temporaryPassword nem a senha gerada.
