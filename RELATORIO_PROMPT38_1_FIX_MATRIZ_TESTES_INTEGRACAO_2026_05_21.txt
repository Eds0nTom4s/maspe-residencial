6) Decisões técnicas
- Foi preferido adicionar `includes` no Surefire para executar `*IT` com `mvn test`, pois o projeto já usa `IT` como padrão para testes de integração e o prompt exige `mvn test` como comando oficial.
- Os testes que dependem de PostgreSQL/Testcontainers continuam “docker-aware” via `@Testcontainers(disabledWithoutDocker = true)` + `Assumptions.assumeTrue(...)`.

7) Riscos restantes
- Em ambientes sem Docker, a suíte `*IT` fica SKIPPED (por design do Testcontainers). Em CI/ambiente com Docker, esses testes devem rodar para validar o comportamento real no PostgreSQL.
- Cobertura de callback AppyPay pós-desativação foi coberta via polling manual (fluxo mais pragmático e estável); callback end-to-end pode ser coberto em fase futura se necessário.
