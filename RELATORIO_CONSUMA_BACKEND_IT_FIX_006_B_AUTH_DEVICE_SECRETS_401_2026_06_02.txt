3. Diagnóstico
- O profile `it-postgres` já tinha `consuma.device.token-hash-secret`, mas não definia explicitamente `consuma.sync.cursor.hmac-secret`.
- Parte relevante dos ITs de device/offline estava usando `DevicePrincipal` injetado via `SecurityMockMvcRequestPostProcessors.authentication(...)`.
- Esse bypass não exercitava:
  - `DeviceAuthenticationFilter`
  - `DeviceAuthService`
  - hash real em `DeviceTokenService.hashToHex(...)`
  - carga real de capabilities
  - semântica real de `Authorization: Device <token>`
- Ao trocar os testes para auth real, surgiu um problema legítimo no bootstrap:
  - `DeviceAuthService` atualiza `lastAuthAt` a cada request
  - isso alterava `updatedAt` da entidade `DispositivoOperacional`
  - `DeviceSyncVersionService.computeBootstrap(...)` usava `disp.updatedAt` no ETag
  - consequência: o bootstrap nunca estabilizava para `304 Not Modified`
- Também apareceu um `ObjectOptimisticLockingFailure` no teste ao alterar `tokenVersion` com uma entidade stale após a autenticação real ter incrementado a versão do dispositivo.

4. Correções aplicadas
- Adicionado helper de suporte real para ITs:

- Esse suporte:

  - ativa dispositivo para teste com token bruto + hash persistido como produção
  - persiste capabilities reais em `device_operational_capabilities`
  - monta header `Authorization: Device <token>`

- Ajuste:
  - o ETag/syncVersion do bootstrap deixou de depender de `disp.updatedAt`
