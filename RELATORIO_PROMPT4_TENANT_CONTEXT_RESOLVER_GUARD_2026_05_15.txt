- Enum: JWT, PLATFORM_ADMIN_SELECTION, QR_TOKEN, DEVICE_TOKEN, PAYMENT_CALLBACK, LEGACY_NONE
- Implementado efetivamente nesta fase: JWT + PLATFORM_ADMIN_SELECTION (outros apenas preparados)

src/main/java/com/restaurante/security/tenant/TenantResolver.java
- Resolve tenant a partir de:
  - Authentication (JWT validado pelo JwtAuthenticationFilter)
  - headers `X-Tenant-Id` / `X-Tenant-Code`
- Regras:
  - Usuário não-admin com 1 TenantUser ATIVO -> auto-resolve
  - Usuário não-admin com múltiplos -> exige header (mas não quebra legado se header ausente)
  - PLATFORM_ADMIN (provisório: ROLE_ADMIN) -> só resolve com header; não valida membership
  - valida tenant ATIVO quando selecionado
- Depende de repos via ObjectProvider para não quebrar testes slice (@WebMvcTest)

src/main/java/com/restaurante/security/tenant/TenantGuard.java
- API de validação contextual (infra):
  - assertTenantActive, assertCurrentTenant, assertCurrentUserBelongsToTenant, assertResourceBelongsToTenant, assertPlatformAdmin, requireContext
- Não aplicado globalmente ainda (uso incremental em fases futuras)

src/main/java/com/restaurante/security/tenant/TenantContextFilter.java
- OncePerRequestFilter que:
  - resolve e seta TenantContext quando possível
  - limpa sempre ao final
- `@ConditionalOnBean(TenantResolver.class)` para não quebrar slices sem resolver

9) DECISÕES TÉCNICAS IMPORTANTES
- Não adicionamos tenantId no JWT nesta fase (evita mudanças invasivas em Auth/JWT).
- Resolver usa banco (`TenantUser`) para inferir tenant quando possível.
- Não aplicamos TenantGuard globalmente em todos endpoints para não quebrar legado.
- ROLE_ADMIN tratado provisoriamente como PLATFORM_ADMIN (até existir role dedicado).

10) RISCOS RESTANTES
- Enquanto endpoints/repositories operacionais não forem tenant-scoped, ainda existe risco de data leakage.
- Usuários autenticados via Atendente/Cliente (principal não persistido em `users`) não resolvem tenant automaticamente nesta fase.
- Validação de roles tenant (TenantUserRole) fica para etapa onde JWT for tenant-aware (claims/authorities por tenant).
