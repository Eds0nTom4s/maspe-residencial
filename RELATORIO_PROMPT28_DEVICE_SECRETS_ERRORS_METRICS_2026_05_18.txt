RELATÓRIO TÉCNICO — PROMPT 28
Hardening de secret por profile + DeviceErrorResponse + métricas de sync/ETag (read-only)
Data: 2026-05-18

1) Resumo executivo
- Implementado fail-fast de configuração sensível em profile `prod` para impedir secrets vazios/curtos/default:
  - `consuma.device.token-hash-secret` (hash HMAC de activationCode/deviceToken)
  - `consuma.sync.cursor.hmac-secret` (HMAC do cursor)
- Padronizados erros de `/device/*` (activate/heartbeat/config/token rotate) com `DeviceErrorResponse` (contrato estável para app POS/KDS).
- Introduzida camada de métricas para sync/auth/heartbeat:
  - interface `DeviceSyncMetricsService`
  - implementação Micrometer quando disponível (sem adicionar dependência obrigatória)
  - fallback NoOp quando Micrometer não está presente.
- Aplicado logging operacional seguro (sem deviceToken/cursor/secrets).

2) Hardening de secrets por profile (prod)
- `src/main/java/com/restaurante/config/SensitiveSecretsValidator.java`
  - Detecta profile `prod` via `Environment.getActiveProfiles()`.
  - Em prod:
    - bloqueia secret vazio
    - bloqueia secret curto (< 32 chars)
    - bloqueia secret default de desenvolvimento (ex: `dev-*-change-me`)
  - Em dev/test:
    - permite, mas loga warning.

3) Properties adicionadas/validadas
- `consuma.device.token-hash-secret` (obrigatório em prod)
- `consuma.sync.cursor.hmac-secret` (obrigatório em prod)

4) DeviceErrorResponse criado
- `src/main/java/com/restaurante/dto/response/DeviceErrorResponse.java`
  - `code`, `message`
  - `recoverable`, `action`
  - `serverTime`
  - `details` sanitizado (opcional)

5) Exception handler device criado
- `src/main/java/com/restaurante/controller/advice/DeviceApiExceptionHandler.java`
  - aplica-se a `/device/*` exceto `/device/sync/*`
  - mapeia 401/403/400 para `DeviceErrorResponse`
  - registra métrica de heartbeat em falhas (quando path = `/device/heartbeat`)

6) Diferença DeviceErrorResponse vs SyncErrorResponse
- `/device/*`: usa `DeviceErrorResponse` (ativação/auth/heartbeat/config/rotate).
- `/device/sync/*`: mantém `SyncErrorResponse` (contrato de sync, cursor, fullSyncRequired).

13) Decisões técnicas tomadas
- “Fail fast” em prod para secrets default/curtos evita hardening tardio em produção.
- Erros do app device precisam contrato próprio (DeviceErrorResponse) separado do sync (SyncErrorResponse).
- Métricas são opcionais: se Micrometer existir no classpath, ativam; caso contrário NoOp.

14) Riscos restantes
- Em produção, recomenda-se bloquear explicitamente o uso de secrets default via CI/infra (e não depender apenas de warning).
- Métricas dependem da presença de Micrometer/observabilidade; sem Actuator, ficam como base interna.
