1) OBJETIVO
================================================================================
Fortalecer o módulo de devoluções/estornos (Prompt 44.1) para fechar lacunas
operacionais previsíveis, adicionando:

- tratamento explícito para WASTE e DO_NOT_RESTOCK;

- cálculo automático de receita/imposto/margem reversos;
- integração formal (placeholder) para eventos de refund/estorno;
- evidência expandida no Evidence Bundle (inventoryEvidence) com métricas, warnings
  e hash determinístico atualizado.

================================================================================
2) ESCOPO IMPLEMENTADO
================================================================================
- Processamento de devolução com diferenciação clara por `restockPolicy`:
  - RESTOCK: gera movimento RETURN_IN e aumenta stock;
  - WASTE: registra retorno como perda (RETURN_WASTE) sem aumentar stock;
  - DO_NOT_RESTOCK: registra COGS_REVERSAL_ONLY (sem stock) e evidencia decisão;
  - MANUAL_REVIEW: bloqueia processamento quando política exige decisão manual.

- Cálculo financeiro reverso no return:
  - totalReturnCost (custo histórico, sem recalcular por averageCost atual);
  - totalRevenueReversed (preferindo INTERNAL_CREDIT_NOTE quando vinculada);
  - totalTaxReversed (preferindo INTERNAL_CREDIT_NOTE quando vinculada);
  - totalMarginReversed = totalRevenueReversed - totalReturnCost;
  - distribuição por linha (revenue/tax/margin) por proporção do “gross” estimado.

- Ponto formal para refund/estorno:
  - evento de domínio `PaymentRefundedForInventoryReturnEvent` (placeholder);
  - listener `InventoryReturnOnPaymentRefundedListener` (sem alterar AppyPay);
  - endpoint tenant/admin para registrar return a partir de refundReferenceId
    (placeholder administrativo).

- Evidence Bundle:
  - `inventoryEvidence` passa a incluir somatórios de reversão financeira e
    métricas/alertas de devoluções com WASTE/DO_NOT_RESTOCK/MANUAL_REVIEW;
  - `returnHash` atualizado para incluir campos financeiros e referências
    (creditNote/refund) sem recalcular bundles antigos.
