Data: 2026-06-11
Branch: backend/ponto-order-acceptance-payment-001

1. Escopo executado
- Consolidação do fluxo backend de pedidos públicos PONTO com identificação obrigatória do cliente.
- Inclusão de aceite, rejeição com motivo e confirmação manual de pagamento pelo tenant.
- Enriquecimento do tracking público e do detalhe/listagem tenant com metadados do fluxo.
- Ajuste do build Docker para imagem multi-stage sem dependência prévia de target/*.jar local.

2. Implementações principais
- Pedido público PONTO agora exige:
  - clienteNome
  - clienteTelefone
  - metodoPagamento
- Métodos manuais CASH/TPA:
  - pedido criado com status financeiro NAO_PAGO
  - ordem manual criada automaticamente
- Métodos digitais APPYPAY/REF:
  - fluxo gateway preservado
  - suporte a pedido direto sem SessaoConsumo no tracking e no start do pagamento
- Novos endpoints tenant:
  - POST /tenant/pedidos/{id}/aceitar
  - POST /tenant/pedidos/{id}/rejeitar
  - POST /tenant/pedidos/{id}/pagamento/manual-confirmar
- Auditoria:
  - eventos de pedido para aceite/rejeição
  - evento operacional para confirmação manual de ordem
