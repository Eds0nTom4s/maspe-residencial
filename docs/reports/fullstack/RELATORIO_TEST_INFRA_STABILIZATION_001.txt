2. Objetivo
-----------
Auditar e estabilizar infraestrutura de validacao E2E/frontend e verify/backend sem alterar regras de produto, UI visual, allowedActions, PedidoOrigem, OperationalTemplatePolicy, pagamento ou implementacoes operacionais novas.

3. Problemas encontrados
------------------------
- E2E falhava ou ficava instavel quando o Playwright nao conseguia acessar o webServer Vite em `127.0.0.1:5173/login` dentro do sandbox de rede.
- Nao havia smoke tenant local suficiente para separar disponibilidade do frontend de backend real.
- Verify amplo falhava em setups de pagamento que desrespeitavam a regra vigente de aceite antes de pagamento.
- Maven emitia logs ruidosos por DEBUG herdado e stacks esperados no console.

4. E2E frontend
---------------
Causa:
- Host/porta/baseURL e disponibilidade do webServer nao estavam suficientemente explicitos para diferenciar falha de webServer, rota, credencial ou backend.

Correcao:
- `playwright.config.ts` centraliza `E2E_HOST`, `E2E_PORT`, `E2E_BASE_URL` e `VITE_API_BASE_URL`.
- Criado helper de sessao tenant e route mocks em `e2e/support/tenantApp.ts`.
- Criado smoke de dashboard, pedidos, caixa e turnos em `e2e/app-smoke.spec.ts`.

Arquivos alterados:
- `playwright.config.ts`
- `e2e/support/tenantApp.ts`
- `e2e/app-smoke.spec.ts`
- `docs/reports/frontend/RELATORIO_TEST_INFRA_STABILIZATION_UI_001.txt`
- `docs/reports/frontend/test-infra-stabilization-001/*`

Comando final:
- `npm run test:e2e`

Resultado:
- PASS fora do sandbox de rede, 6 passed, 1 skipped, 14.7s.

5. Playwright/webServer
-----------------------
Config anterior:
- Dependencia maior dos defaults do Vite e ausencia de `--strictPort` no comando do webServer.

Problema:
- Em falha, o Playwright parava antes de deixar claro se a causa era porta, rota, credencial, backend ou isolamento de rede.

Config final:
- `npm run dev -- --host ${E2E_HOST:-127.0.0.1} --port ${E2E_PORT:-5173} --strictPort`.
- URL aguardada: `/login`.
- timeout: 120s.
- `reuseExistingServer` habilitado somente fora de CI.

Resultado:
- WebServer deterministico; conflito de porta nao vira troca silenciosa.
