--------------------------------------------------------------------------------
5. LÓGICA DE SEGURANÇA
--------------------------------------------------------------------------------

  OwnerActionTokenExtractor:
  - Header > body > null (nunca query param)
  - Lança OWNER_ACTION_TOKEN_REQUIRED se ausente e required=true
  - Lança OWNER_ACTION_TOKEN_QUERY_PARAM_NOT_ALLOWED se em query param

  SessaoParticipanteOwnerTokenActionService.resolveAndValidate():
  1. resolveQrContext(qrToken) → tenantId + sessaoId
  2. ownerTokenService.validateAndUse() → SessaoOwnerActionToken + owner entity
  3. owner.status == ACTIVE → ou OWNER_ACTION_TOKEN_OWNER_NOT_ACTIVE
  4. owner.role == OWNER  → ou OWNER_ACTION_TOKEN_OWNER_NOT_OWNER

  Auditoria sanitizada:
  - Metadados NUNCA contêm rawToken, tokenHash ou ownerActionToken
  - Falhas de auditoria são absorvidas (log.warn), não propagam

--------------------------------------------------------------------------------
6. REVOGAÇÃO AO FECHAR SESSÃO
--------------------------------------------------------------------------------

  Método: SessaoOwnerActionTokenService.revokeActiveTokensBySessao()
  - PESSIMISTIC_WRITE: evita race condition entre close e uso simultâneo
  - Idempotente: retorna 0 se já não há tokens ACTIVE
  - Auditoria agregada: revokedCount, reason, sessaoId — sem hashes
  - Falha de auditoria é absorvida (log.warn)

  Integração esperada em SessaoConsumoService.fechar():
    ownerTokenService.revokeActiveTokensBySessao(tenantId, sessaoId, "SESSION_CLOSE", ip, ua);
